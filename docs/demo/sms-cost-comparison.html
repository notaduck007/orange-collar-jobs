⚠️

A2P 10DLC Registration Required (US)

All US SMS sent from an application ("Application-to-Person") require 10DLC brand and campaign registration since 2021. This applies regardless of provider. Failure to register causes message filtering by carriers. One-time setup: ~$20 brand fee + ~$10/month per campaign.

⚡

Verify vs. Messages API — Which for OTP?

Use Twilio Verify for all authentication OTPs. It costs $0.05/verification but eliminates ~50 lines of NestJS code (code generation, Redis TTL, retry throttling, brute-force protection) and handles carrier deliverability automatically. At 10,000 OTPs/month the cost is $500 — justified by engineering time saved and reduced fraud risk.

If cost is a priority at scale, switch OTPs to the raw Messages API ($0.0079/SMS) and implement the OTP logic yourself in NestJS with Redis. The SmsService example in the Integration section shows both approaches.

🔌NestJS Integration

Twilio integrates directly into the NestJS auth domain. Install once, use for all three SMS channels.

Install

bun add twilio

import { Module } from '@nestjs/common';

SmsService — full implementation (src/core/sms/sms.service.ts) 

import { Injectable, Logger } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import Twilio from 'twilio';           // ← official SDK — types bundled, no @types needed

@Injectable()
export class SmsService {
  private readonly client: Twilio.Twilio;
  private readonly from: string;
  private readonly verifyServiceSid: string;
  private readonly logger = new Logger(SmsService.name);

  constructor(private readonly config: ConfigService) {
    this.client = Twilio(
      config.getOrThrow<string>('TWILIO_ACCOUNT_SID'),
      config.getOrThrow<string>('TWILIO_AUTH_TOKEN'),
    );
    this.from             = config.getOrThrow<string>('TWILIO_FROM_NUMBER');
    this.verifyServiceSid = config.getOrThrow<string>('TWILIO_VERIFY_SERVICE_SID');
  }

  // ── Twilio Verify API ── (Authentication channel) ─────────────────────────
  // Product: twilio.com/verify · $0.05/verification · first 100/month free
  // Use for: register, login, password reset OTPs

  /** Send a 6-digit OTP via Twilio Verify (SMS, WhatsApp, or email) */
  async sendVerificationCode(to: string, channel: 'sms' | 'whatsapp' | 'email' = 'sms'): Promise<void> {
    await this.client.verify.v2
      .services(this.verifyServiceSid)
      .verifications.create({ to, channel });
    this.logger.log(`Verification code sent via ${channel} to ${to}`);
  }

  /** Check the OTP the user entered — returns true if correct */
  async checkVerificationCode(to: string, code: string): Promise<boolean> {
    const result = await this.client.verify.v2
      .services(this.verifyServiceSid)
      .verificationChecks.create({ to, code });
    return result.status === 'approved';
  }

  // ── Twilio Lookup API ── (Phone validation) ────────────────────────────────
  // Product: twilio.com/lookup · $0.005/lookup · call before sendVerificationCode
  // Use for: validate phone on registration / quick-apply form

  /** Validate a phone number. Returns false for landlines, invalid numbers. */
  async isValidMobileNumber(phone: string): Promise<boolean> {
    try {
      const lookup = await this.client.lookups.v2.phoneNumbers(phone).fetch({
        fields: 'line_type_intelligence',
      });
      const lineType = lookup.lineTypeIntelligence?.type;
      return lineType === 'mobile' || lineType === 'nonFixedVoip';
    } catch {
      return false; // invalid number format
    }
  }

  // ── Twilio Programmable SMS ── (Notifications channel) ────────────────────
  // Product: twilio.com/sms · $0.0079/SMS (US) · pay-as-you-go
  // Use for: application status updates, new applicant alerts, job alerts

  /** Notify seeker when their application status changes */
  async sendApplicationUpdate(
    to: string,
    jobTitle: string,
    status: string,
  ): Promise<void> {
    await this.client.messages.create({
      to,
      from: this.from,
      body: `WarehouseJobs: Your application for "${jobTitle}" is now ${status}. View: warehousejobs.com/applications`,
    });
  }

  /** Notify employer of a new applicant */
  async sendNewApplicantAlert(
    to: string,
    jobTitle: string,
    applicantName: string,
  ): Promise<void> {
    await this.client.messages.create({
      to,
      from: this.from,
      body: `WarehouseJobs: ${applicantName} applied for your "${jobTitle}" posting. Review: warehousejobs.com/employer/applications`,
    });
  }

  /** Send opt-in job alert SMS (BullMQ worker calls this per seeker) */
  async sendJobAlert(to: string, jobTitle: string, city: string, applyUrl: string): Promise<void> {
    await this.client.messages.create({
      to,
      from: this.from,
      body: `WarehouseJobs: New job alert — ${jobTitle} in ${city}. Apply in 60s: ${applyUrl}\nReply STOP to unsubscribe.`,
    });
  }
}

🏆Recommendation

Recommended Architecture: Twilio Primary

One provider, one SDK, all three SMS channels covered. Simple to integrate, scales linearly, pay-as-you-go with no monthly minimums.

Auth OTP

Twilio Verify API — $0.05/verification. Handles code generation, delivery, expiry, and retry limits. Removes OTP state management from NestJS. At 10,000 verifications/month = $500/month, or fall back to manual OTP via standard Twilio Messages API at $0.0079/SMS = $79/month.

Application Notifications

Twilio Messages API — $0.0079/SMS via same SDK and phone number. No additional setup. BullMQ worker sends notifications asynchronously. At 20,000/month = $158/month.

Job Alerts (Phase 2)

Twilio Messages API initially — same infrastructure, zero additional setup. At scale (>50k alerts/month), evaluate adding Klaviyo SMS for segmentation, opt-in management, and combined email+SMS automation. Not required until late Phase 3.

Total Cost Estimates

Startup: ~$30–40/month
Growth (30k SMS): ~$243/month
Scale (150k SMS): ~$886–$1,191/month

No monthly minimums. No contract required to start.

🗓

When to Add Klaviyo SMS

Add Klaviyo as a secondary provider (job alerts only) when: (a) monthly job alert volume exceeds 30,000 SMS, AND (b) you need audience segmentation by job category, location, or seeker profile — which Klaviyo handles significantly better than custom Twilio logic. Until then, Twilio alone is simpler and cheaper.
