⚖️Key Technical Decisions

Database Self-hosted PostgreSQL 16 Portability; full Prisma control; no Supabase vendor lock-in; long-term scalability
Auth Custom NestJS JWT (HS256) Full token/role/refresh control; supports migration from Supabase Auth
File Storage MinIO → Cloudflare R2 S3-compatible; same code path in dev and prod; R2 zero egress fees; enterprise-scale
Batch Ingestion Sync ≤100 / BullMQ async >100 Avoids HTTP timeouts on large feeds; reliable retry; dedup by externalId
Frontend Migration Big Bang (Phase 7) Avoids auth session split-brain between Supabase and NestJS
ORM Prisma 5 Type-safe schema; migration history; excellent TypeScript integration
Monorepo Bun workspaces (src/api/) Self-contained NestJS app; extractable at any time with no repo-wide consequences

🔐Authentication Flow

1.  POST /api/v1/auth/login
    → { accessToken (15 min), refreshToken (30 days) }

2.  Every request:
    Authorization: Bearer {accessToken}

3.  On 401 (token expired):
    POST /api/v1/auth/refresh  { refreshToken }
    → new accessToken + rotated refreshToken

4.  Wrong role → HTTP 403
    No token   → HTTP 401

⚡Quick Apply & Application Pipeline

Unauthenticated Apply

Minimum required: name + phone + ZIP. Resume is always optional. Designed for mobile workers in the field. Target: under 60 seconds, under 3s server time.

Authenticated Apply

Seeker profile snapshot auto-filled from JWT. Screening question answers validated. Interview slot booking is atomic (no double-booking).

Application State Machine

new → reviewing → shortlisted → interview → offer → hired ↘ ↗ → rejected (employer action, from any state) → withdrawn (seeker action, from any state) Invalid transition → HTTP 400 with transition path in error details
